# Contributing to DrawWithFriends

Thank you for considering contributing to DrawWithFriends! This document provides guidelines and information for contributors.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR-USERNAME/DrawWithFriends.git`
3. Create a new branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test your changes thoroughly
6. Commit your changes: `git commit -m "Add your feature"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Setup

### Prerequisites
- macOS 12.0 or later
- Xcode 15.0 or later
- Swift 5.0 or later

### Building the Project
1. Open `DrawWithFriends.xcodeproj` in Xcode
2. Select your target device or simulator
3. Press Cmd+R to build and run

## Code Style

- Follow Swift naming conventions
- Use SwiftUI for UI components
- Write clear, self-documenting code
- Add comments for complex logic
- Keep functions focused and single-purpose

## Pull Request Guidelines

- Provide a clear description of the changes
- Include screenshots for UI changes
- Ensure the code builds without warnings
- Test on both iPhone and iPad simulators
- Update documentation if needed

## Feature Ideas

Some features we'd love to see:
- [ ] Color picker for different brush colors
- [ ] Brush size selector
- [ ] Real-time collaboration via network
- [ ] Save and load drawings
- [ ] Share drawings as images
- [ ] Undo/Redo functionality
- [ ] Different brush styles
- [ ] Eraser tool
- [ ] Background patterns or images

## Reporting Bugs

When reporting bugs, please include:
- iOS version
- Device or simulator used
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots if applicable

## Questions?

Feel free to open an issue for any questions or discussions!
